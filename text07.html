جلسه 7

اجرا و دیباگ برنامه در PyScripter:

  • برای اجرای برنامه در PyScripter می‌توان از دکمه درنوار ابزار، یا گزینه Run در منوی Run استفاده کرد. کلید میانبر آن نیز Ctrl + F9 است.
  • برای دیباگ کردن برنامه و دنبال کردن اجرای خطوط برنامه از گزینه Step Into در منوی Run استفاده می‌کنیم که کلید میانبر آن F7 است. با هر بار فشردن کلید F7 می‌توانیم اجرای برنامه را خط به خط پیش ببریم.
  • در هنگام دیباگ در قسمت Variables در پایین صفحه می‌توانیم مقدار متغیرهای برنامه را مشاهده کنیم.
  • مشاهده مقدار متغیرها با نگهداشتن اشاره‌گر ماوس روی آنها در متن برنامه نیز امکان‌پذیر است.
  • در قسمت Watches در پایین صفحه می‌توان متغیرها یا عبارتهایی را وارد کرد و هنگام دیباگ مقدار آنها را مشاهده کرد.
  • گزینه Run to Cursor در منوی Run برنامه را تا خطی که اشاره‌گر صفحه کلید در آن قرار دارد، اجرا می‌کند.
  • گزینه Step Out در منوی Run باعث می‌شود تابع فعلی تا انتها اجرا شود.
  • گزینه Step Over در منوی Run باعث می‌شود اگر فراخوانی تابع داشته باشیم، بدون ورود به تابع، آن را اجرا کنیم و به خط بعدی برویم.
  • با کلیک در نوار سمت چپ یک خط برنامه می‌توان یک breakpoint قرار داد یا آن را حذف کرد. گزینه Toggle breakpoint در منوی Run هم همین کار را انجام می‌دهد.
  • گزینه Debug در منوی Run باعث می‌شود اجرای برنامه تا رسیدن به اولین breakpoint ادامه پیدا کند. هنگام دیباگ کردن نام این گزینه به Resume تغییر می‌کند.
  • برای توقف دیباگ برنامه در PyScripter می‌توان از دکمه درنوار ابزار، یا گزینه Abort Debugging در منوی Run استفاده کرد.

ماژول یک فایل شامل تعریف هایی مانند تابعها است که به منظور استفاده در دیگر برنامه‌های پایتون نوشته شده است. برای استفاده از یک ماژول باید آن را import کنیم. برای این کار روشهای مختلفی وجود دارد:
روش اول 

import math
print(math.sqrt(4))
روش دوم: تعیین نام مستعار 
import math as mh
print(mh.sqrt(4))
روش سوم: فقط import موارد لازم، نیاز به نوشتن نام ماژول هنگام استفاده نیست 
from math import sqrt, sin
print(sqrt(4))
print(sin(math.pi))
روش چهارم: import همه چیز، نیاز به نوشتن نام ماژول هنگام استفاده نیست 
from math import *
print(sqrt(4))
print(sin(pi))

برای ساختن یک ماژول کافی است آن را در یک فایل با پسوند py ذخیره کنیم. مثلاً در فایل L7_even.py یک تابع به نام iseven و یک متغیر به نام a_var تعریف شده است و می‌خواهیم از این فایل به عنوان یک ماژول استفاده کنیم. در فایل L7_import.py ماژول L7_even را import کرده‌ایم و از تابع iseven و متغیر a_var استفاده کرده‌ایم.

در هر برنامه پایتون یک متغیر از پیش تعریف شده به نام __name__ وجود دارد. اگر خود برنامه اجرا شود این متغیر برابر رشته '__main__' است.
اما اگر برنامه به صورت یک ماژول در برنامه دیگری import شود، این متغیر حاوی نام فایل خواهد بود.

یک فضای نام (namespace) مجموعه شناسه‌هایی است که به یک ماژول یا تابع (یا کلاس) تعلق دارند. مثلاً در دو ماژول متفاوت می‌توان متغیرهایی با نام یکسان داشت که هیچ ارتباطی با هم ندارند زیرا هر ماژول فضای نام خود را دارد. به طور مشابه در دو تابع متفاوت می‌توان متغیرهایی با نام یکسان داشت که هیچ ارتباطی با هم ندارند زیرا هر تابع فضای نام خود را دارد.

حوزه (scope) برای یک شناسه، بخشی از برنامه است که در آن می‌توان از آن شناسه استفاده کرد.

  • حوزه محلی (local) شامل موارد تعریف شده درون یک تابع است.
  • حوزه سراسری (global) شامل موارد تعریف شده درون یک فایل یا ماژول است.
  • حوزه از پیش تعریف شده (built-in) شامل موارد تعریف شده توسط پایتون است که بدون import ماژولی قابل استفاده هستند مانند min یا range.
چون ممکن است در جاهای مختلف متغیرها یا تابعهای هم نام تعریف شوند، باید یک اولویت بندی وجود داشته باشد. اولویت در پایتون به ترتیب با حوزه محلی، سپس حوزه سراسری، و در آخر حوزه از پیش تعریف شده است. همچنین الویت با موارد تعریف شده نزدیک‌تر است.
مثال: اگر فایل m1 حاوی a = 1 و فایل m2 حاوی a = 2 باشد. خروجی برنامه زیر چیست؟ 
from m1 import *
from m2 import *

print(a)
a = 3
print(a)
